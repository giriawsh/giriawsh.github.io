《JS权威指南》第二章

2020-09-14 Views JavaScript权威指南学习系列 | 学习系列 | JavaScript | JS学习系列2014字8 min read

✍️词法结构

对于每项编程语言，都有其作为基准的词法结构，包括字符集的设定、注释约定、变量命名规则、标识符和保留字、解析规则等。这里记录一下各个小节中我学习中认为比较重要的部分。

1. 字符集

JS使用的是Unicode字符集。Unicode是ASCII和Latin-1的超集（父集）

👉Unicode是什么?

世界上存在着多种编码方式，同一个二进制数字可以被解释成不同的符号。因此，要想打开一个文本文件，就必须知道它的编码方式，否则用错误的编码方式解读，就会出现乱码。

可以想象，如果有一种编码，将世界上所有的符号都纳入其中。每一个符号都给予一个独一无二的编码，那么乱码问题就会消失。这就是 Unicode，就像它的名字都表示的，这是一种所有符号的编码。

Unicode 当然是一个很大的集合，现在的规模可以容纳100多万个符号。每个符号的编码都不一样，比如，U+0639表示阿拉伯字母Ain，U+0041表示英语的大写字母A，U+4E25表示汉字严。具体的符号对应表，可以查询unicode.org，或者专门的汉字对应表。

👉ASCII是什么？

如上所说，ASCII是unicode的一个子集。计算机内部，所有信息最终都是一个二进制值。每一个二进制位（bit）有0和1两种状态，因此八个二进制位就可以组合出256种状态，这被称为一个字节（byte）。也就是说，一个字节一共可以用来表示256种不同的状态，每一个状态对应一个符号，就是256个符号，从00000000到11111111。

上个世纪60年代，美国制定了一套字符编码，对英语字符与二进制位之间的关系，做了统一规定。这被称为 ASCII 码，一直沿用至今。

ASCII 码一共规定了128个字符的编码，比如空格SPACE是32（二进制00100000），大写的字母A是65（二进制01000001）。这128个符号（包括32个不能打印出来的控制符号），只占用了一个字节的后面7位，最前面的一位统一规定为0。

这个符号表相信学过计算机导论都有见到过它的身影。

但是用来表示其他语言，128个符号是不够的。一些欧洲国家就决定，利用字节中闲置的最高位编入新的符号进行拓展，可以表示最多256个符号。

👉Latin-1字符集

SO Latin-1字符集是Unicode字符集的一个子集，对应于Unicode字符指令表的前256个条目。其中Unicode字符为双字节16位，可以表示任何一种语言符号；而Latin-1字符集是单字节8位，只能够表示英文和西欧字符。

因为ISO-8859-1编码范围使用了单字节内的所有空间，在支持ISO-8859-1的系统中传输和存储其他任何编码的字节流都不会被抛弃。**换言之，把其他任何编码的字节流当作ISO-8859-1编码看待都没有问题。**这是个很重要的特性，MySQL数据库默认编码是Latin1就是利用了这个特性。ASCII编码是一个7位的容器，ISO-8859-1编码是一个8位的容器。

latin-1 这个字符集的牛逼之处、latin-1字符集是在ascii码上的一个扩展，它把ascii码没有用到过的字节码都给编上了对应的字符，所以它能表示的字符就更多了；针对单个字节来说就没有它不能解码的。

1.1 大小写区分

JS是区分大小写的，但是HTML不区分大小写（尽管XHTML区分大小写）。

例如：在HTML设置事件处理程序时 onclick = onClick

但是在JS（或者XHTML中）必须使用onclick

1.2 空格、换行符和格式控制符

JS会忽略程序中标识之间的空格。（标识的话学过编译原理应该都知道，是词法分析的解析出的单位，本人写过一个基于c++的词法分析器，仅作为校园课程作业参考：Clex词法分析器）也是由于代码中可以随意使用空格和换行，实际开发中需要进行编码风格的统一，提高代码可读性。

同等会被解析成空格的字符有：普通空格符（\u0020）、水平制表符（\u0009）、垂直制表符（\u0008）、换页符（\000C）等所有Unicode中的zs型字符（没有标志符号，也不用来控制或者格式表征的字符）

会被解析成换行符的有：换行符（\u000A）、回车符(\u000D)、行分隔符（\u2028）、段分隔符（\u2029）。回车加换行会被解析成一个单行结束符

格式控制符便是Unicode中的Cf类，控制文本的视觉显示（例如从左到右or从右到左）

1.3 Unicode转义序列

为了更好的兼容性，JS定义了一种特殊序列，使用6个ASCII字符来代表人已16位Unicode内码，以\u为前缀，其后跟随4个十六进制数。

这种转移写法可以用在JS字符串直接量、正则表达式直接量、标识符（关键字除外）中。

Unicode转义写法也可以出现在注释中，但是只被当做上下文中的ASCII字符处理，并不会被解析为其对应的Unicode字符。

1.4 标准化

Unicode标准为所有字符定义了一个首选的编码格式，并给出了一个标准化的处理方式，将文本转换为一种合适比较的标准格式。

JS在处理时会认为它正在解析的程序代码已是这种标准格式，不会再做标准化处理。

2. 标识符和保留字

2.1 标识符

标识符：就是一个名字

必须以字母、下划线、开始，后续可以是字母、数字、下划线、开始，后续可以是字母、数字、下划线、开始，后续可以是字母、数字、下划线、（数字不允许打头）

JS允许标识符中出现Unicode字符全集中的字母和数字，但是为了可移植性与便于书写的考虑，通常只使用ASCII字母和数字书写标识符

3. 可选的分号

这一章介绍了许许多多奇奇怪怪的偷懒不加分号的解释法……

大致规则如下：

  • JS只有在缺少了分号就无法正确解析代码的时候，JS才会填补分号（像不像语法翻译里面的自动错误纠正，实在解析不动了才报错然后再弹出看不懂的，再继续解析）。但有两个例外。
  • 第一个例外是在涉及return、break、continue时，如果紧跟换行，则会在换行处填补分号。
return
true;
 ==>
return; true;
  • 第二个例外是++和--运算符时，将会作为下一行代码的前缀操作符与之一起解析（没想到吧老弟，我跟身后的友军一伙了）
x
++
y
==>
x; ++y;

这可能和这俩运算符本身的结合性有关。
