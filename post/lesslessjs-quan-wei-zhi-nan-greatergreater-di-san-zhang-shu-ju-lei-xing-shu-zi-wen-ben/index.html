《JS权威指南》第三章1——数据类型、数字、文本

2020-09-16 Views JavaScript权威指南学习系列 | 学习系列 | 前端 | JavaScript | JS学习系列3149字12 min read

✍️前言

本章会分多篇介绍JS中的类型、值和变量，后续还会有相应的章节进行深入介绍。

1. 基本概念

1.1 数据类型

数据类型：能够表示并操作的值的类型

编程语言最基本的特性就是能够支持多种数据类型。

变量：一个值的符号名称，可以通过名称来获得对值的引用

JS的数据类型：

  • 原始类型：包括数字、字符串、布尔值、null、undefined，后面两个分别代表了各自特殊类型的唯一成员（孤儿确认）。
  • 对象类型：除上面以外的所有东西。
    • 对象时属性的集合，每个属性都由“名/值对”构成。其中有一个比较特殊的对象——全局对象，后面会讲到。
    • 包括一般对象、数组、函数等

数组：普通的JS对象是“命名值”的无序集合。JS同样定义了一种特殊对象即数组，表示带编号的值的无序集合。

JS为数组定义了专用的语法，使数组拥有一些和普通对象不同的特有行为特性。（数据类型中的上层阶级确认）

函数：是具有与它相关联的可执行代码的对象。（爷有关联可执行代码）

通过调用函数来运行可执行代码，并返回运算结果。和数组一样，函数的行为特征也与众不同，JS也为它定义了专用语法。（上层阶级+1）

最重要的是，函数们都是真值（我永远是true理）。并且JS可以将它们当做普通对象来对待（皇帝不高兴，把你当下人也是可以的）。

构造函数：当函数用来初始化（new）一个新建的对象

每个构造函数定义了一类对象——由不同构造函数初始化的对象的集合。

类可以看做是对象类型的子类型（毕竟是由对象类型的子集->函数->用来初始化对象->构造函数->定义了某类），包括：

  • 数组类
  • 函数类
  • 日期类
  • 正则类
  • 错误类
  • 自己定义的构造函数类

1.2 内存管理机制

JS解释器有自己的内存管理机制，可以自动对内存进行垃圾回收。程序员只要按需创建对象，不用担心销毁和内存回收工作。

当不再有任何引用指向一个对象，解释器就会自动回收它所占用的内存资源。

1.3 其他划分

根据能不能拥有方法

  • 可变mutable类型
    • 值可以修改
    • 对象、数组
  • 不可变immutable类型
    • 不可以修改
    • 数字、字符串（没错我也不行）、布尔值、null、undefined

1.4 类型转换

JS可以自由地进行数据类型转换，但是转换规则会对“判断相等”造成影响

1.5 类型定义

JS变量是无类型的（untyped），变量可以被赋予任何类型的值（是一种很酷的设计，为自己的可移植性、灵活性奠定了基础），同样一个变量也可以被重新赋予不同类型的值。

JS采用词法作用域。

  • 作用域：是指程序源代码中定义变量的区域

  • 词法作用域：函数的作用域在函数定义的时候就决定了。

  • 动态作用域：函数的作用域是在函数调用的时候才决定的

全局变量：不在任何函数内声明的变量，它在JS程序中的任何地方都是可见的。

函数作用域：在函数内声明的变量具有的作用域，只在函数内可见

这些下文也会讲到。

2. 数字

JS不区分整数值和浮点数值，JS中所有数字均用浮点数值表示。

JS采用IEEE754标准定义的64位浮点格式表示数字。

*IEEE754标准

在IEEE 754标准中浮点数由三部分组成：符号位（sign bit），有偏指数（biased exponent），小数（fraction）。浮点数分为两种，单精度浮点数（single precision）和双精度浮点数（double precision），它们两个所占的位数不同。

  • 单精度浮点数（共32位）：
    1个符号位
    8个指数位
    23个小数位
  • 双精度浮点数（共64位）-> JS属于这一种
    1个符号位
    11个指数位
    52个小数位

2.1 整数范围

整数范围是从-253 ~253（包含边界值）

如果超过此范围的整数，则无法保证低位数字的精度。

然而JS中实际操作（例如数组索引、位操作符）是基于32位整数

数字直接量：当一个数字直接出现在JS程序中时

2.2 整型直接量

在JS程序中，用一个数字序列表示一个十进制整数。

除了十进制，JS还能识别十六进制（以0x或0X为前缀），JS的某些实现可以允许采用八进制（以0开始）

但是有些不支持八进制，因此最好不要使用以0位前缀的整型直接量。（ES6的严格模式下，八进制直接量是被禁止的）

2.3 浮点型直接量

支持传统实数写法：一个实数 = 整数部分 + 小数点 + 小数部分

指数计数法： 实数后+e/E+(正号或负号)+整型的指数

eg: 1.473821E-32 = 1.473821*10-32

2.4 JS中的算术运算

基本运算符：+ - * / %

复杂运算：由Math对象的属性定义的函数和常量来实现

溢出和下溢

溢出：当数字运算结果超过了JS所能表示的数字上下限

返回Infinity和-Infinity

基于他们的加减乘除运算结果还是无穷大（保留正负号）

下溢：当运算结果无限接近于零并比JS所能表示的最小值还小

返回0.当一个负数下溢是会返回负零

特殊运算情况

  • 被零整除：不报错，返回Infinity或-Infinity
  • 0/0：没有意义，返回NaN
  • 无穷大除以无穷大，给任意负数作开方运算，运算符与不是数字或无法转换为数字的操作数一起使用：返回NaN

Inifity和NaN和Number对象定义的属性值在ES5中被定义为只读。

非数字值

非数字值：NaN、字符串、对象等不是数字的值

它和任何值都不相等，包括自身。

所以不能通过x==NaN判断是否是NaN,应该通过x!=x来判断，也可以使用函数isNaN()：如果参数是NaN或者一个非数字值都返回true

还有一个类似函数isFinite()：参数不是NaN、Infinity或-Infinity或非数字值的时候返回true

经过测试有两种很有意思的情况（图中的第二个和第三个红框）

我们看下MDN中该函数的定义：

isFinite (number)

Returns false if the argument coerces to NaN, +∞, or −∞, and otherwise returns true.

If ToNumber(number) is NaN, +∞, or −∞, return false.

Otherwise, return true.

实际上针对输入，该函数隐式调用了Number()转换函数，在内部实现中也可以看到：

function isFinite(someInput) {
  return !isNaN(someInput) &&
    someInput != Number.POSITIVE_INFINITY &&
    someInput != Number.NEGATIVE_INFINITY;
}

注意使用的是!=包含了比较时的强制类型转换。再到浏览器里试验一下。

真相大白了，经过Number转换后，[]和[1]和null返回的都不是NaN或者Infinity。

负零值

它和正零值是相等的（甚至在严格相等测试下也是），这两个值几乎一模一样，除了作为除数。

2.5 二进制浮点数和四舍五入错误

实数有无数个，但JS通过浮点数表示是有限的。当JS中使用实数的时候，常常知识真实值的一个近似表示。

IEEE-754浮点数表示法可以准确地表示二幂次分数（1/2,1/8等），但并不能准确地表示类似0.1这样的简单数字

由此带来0.1 + 0.2 !== 0.3的问题，原因如下：

首先，计算机无法直接对十进制的数字进行运算，这是硬件物理特性已经决定的。这样运算就分成了两个部分：先按照IEEE 754转成相应的二进制，然后对阶运算。精度损失可能出现在进制转化和对阶运算过程中*

  • 进制转换

0.1和0.2转换成二进制后会无限循环

0.1 -> 0.0001100110011001...(无限循环)
0.2 -> 0.0011001100110011...(无限循环)

但是由于IEEE 754尾数位数限制，需要将后面多余的位截掉

这样在进制之间的转换中精度已经损失

  • 对阶运算

由于指数位数不相同，运算时需要对阶运算 这部分也可能产生精度损失，最后的结果是

0.0100110011001100110011001100110011001100110011001100

结果转换成十进制之后就是0.30000000000000004

因此在数值比较中，可以使用ES6中引入的极小的常量Number.EPSILON。它表示 1 与大于 1 的最小浮点数之间的差。

Number.EPSILON实际上是 JavaScript 能够表示的最小精度。误差如果小于这个值，就可以认为已经没有意义了，即不存在误差了。

Number.EPSILON可以用来设置“能够接受的误差范围”。比如，误差范围设为 2 的-50 次方（即Number.EPSILON * Math.pow(2, 2)），即如果两个浮点数的差小于这个值，我们就认为这两个浮点数相等。

0.1 + 0.2 - 0.3
// 5.551115123125783e-17
5.551115123125783e-17.toFixed(20)
// '0.00000000000000005551'
0.1 + 0.2 === 0.3 // false
5.551115123125783e-17 < Number.EPSILON * Math.pow(2, 2)
// true

JS的未来版本或许会支持十进制数字类型以避免舍入问题，当前可以更经常地使用大整数进行重要金融计算（譬如用“分”来代替小数“元”等）

2.6 日期和时间

使用Date()构造函数，用来创建表示日期和时间的对象。提供了一些简单的API。

3. 文本

字符串：一组由16位值组成的不可变的有序序列，每个字符通常来自于Unicode字符集。字符串的长度是所含16位值得个数。

那些不能表示为16位的Unicode字符则遵循UTF-16编码规则——用两个16位值组成的一个序列表示。

也就是说，一个长度为2的JS字符串有可能表示一个Unicode字符。

3.1 字符串的索引

从零开始。空字符串的长度为0.

3.2 字符串直接量

单引号可以包含双引号，双引号也可包含单引号。

字符串可以用反斜线来拆分数行。

  • 用单引号定界字符串：

注意使用反斜线进行转义撇号

  • JS代码与HTML代码混杂时：

最好分开使用各自的引号风格。譬如在JS中用单引号，在HTML中使用双引号。

<button onclick="alert('Thank you')">Click Me</button>

3.3 转义字符

反斜线符号后加一个字符，就不再表示它们的字面含义了。

比较有意思的是转义字符的英文——escape sequence，加上反斜线老子就逃跑不受你原本编译控制了。

3.4 字符串的使用

+ 用于数字，表示两数相加，用于字符串表示字符串拼接

length属性可以得到字符串的长度

书中还提到了许多调用方法，不过也可以查看后面的大表（不愧是你万能工具书）

记住！！在JS中字符串是固定不变的，这些方法实际上都返回了新的字符串，原字符串本身并没有发生改变。

ES5中，字符串可以当做只读数组，除了使用charAt()，也可以用一般的括号方法访问

s = "hello"
s[0] // => "h"
s[s.length-1] // => "o"

3.5 字符串中的模式匹配

正则表达式：JS定义了RegExp()构造函数，用来创建表示文本匹配模式的对象。JS采用Perl的正则表达式语法。（除了Perl还有Posix标准，可自行搜索）
